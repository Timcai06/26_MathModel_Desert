你是一名全国大学生数学建模竞赛教练、运筹优化研究者和博弈论研究者。请对 2020 年高教社杯数学建模竞赛 B 题“穿越沙漠”进行一次可复现、可审计、面向竞赛论文落地的深入研究。目标不是泛泛讲思路，而是形成“题面事实—数学模型—算法—实验—结论—论文表达”完整证据链。

## 一、证据与事实核验

1. 必须先逐页阅读我上传的题面、附件和 Result 模板，不得仅依赖网络上的二手解答。
2. 逐项整理并核对六关的地图邻接关系、起点、终点、村庄、矿山、截止日期、初始资金、负重、资源重量与价格、三种天气消耗、矿山收益、已知天气和玩家人数。
3. 地图邻接只按公共边界判断，共顶点不算相邻。把六关地图转成邻接表并做以下自动检查：节点齐全、无自环、邻接对称、图连通、特殊节点存在、题面标注可回溯。
4. 对 PDF 图像、DOCX 文本、网络资料之间的差异单列“证据冲突表”。任何无法确定的规则必须标为假设，说明它会如何影响结论。
5. 联网资料优先使用官方题面、官方附件、公开论文或可运行的开源代码。引用必须给出可点击链接、发布日期或访问日期，并区分“原始规则”“他人做法”“本研究推导”。不得把公开答案当成最优性证明。

## 二、统一游戏模型

请建立一个统一的离散时间状态模型。至少定义：

- 状态：日期、位置、现金、水、食物、已观测天气、其他玩家可观测信息；
- 行动：相邻移动、原地停留、挖矿、村庄采购；
- 状态转移：晴朗/高温/沙暴下的基础消耗，移动 2 倍，挖矿 3 倍；
- 约束：整数箱、每日负重不超过上限、现金非负、沙暴不可移动、到矿当日不可挖矿、未到终点前任一资源耗尽即失败、截止日前到达终点；
- 结算：终点剩余物资按基准价一半退回；
- 记录口径：Result 表记录当日消耗后、若有采购则采购后的状态。

多人规则必须按题面精确处理：若同一天有 k 名玩家走同一条 A→B 边，每人的消耗为基础消耗的 2k 倍；若 k 人在同一矿山挖矿，每人消耗仍为基础消耗的 3 倍、收益为基础收益的 1/k；若 k 人在同一村庄采购，每箱价格为基准价 4 倍；其他情况按单人规则。明确同时行动、信息披露和策略承诺的时间顺序。

先给出符号表、状态转移方程、目标函数和可行域，再写算法。所有关键公式要解释经济含义和量纲。

## 三、问题1：天气全知的单人全局最优策略

1. 给出可推广的一般模型，优先比较以下两类精确方法：
   - 时间扩展网络上的混合整数规划；
   - 带资源与现金标签、支配剪枝的动态规划/最短路。
2. 说明如何处理村庄采购、终点残值、提前到达、相同最终收益的多重最优解。
3. 分别精确求解第一关和第二关。必须报告：求解器状态、最优值、最优性间隙、计算时间、初始采购、逐日位置/行动/采购/现金/水/食物。
4. 用一个与求解器独立的逐日模拟器重新执行最优策略；逐日验证相邻移动、天气、消耗、负重和现金。
5. 自动回填 Result.xlsx，并检查 Excel 中每个数都等于模拟器流水。
6. 以下数值只作为回归基准，不得代替独立求解：第一关已知可达到 10470 元，第二关已知可达到 12730 元。若结果不同，优先排查天气抄录、沙暴挖矿、村庄采购时点、终点残值和“到矿当日不能挖矿”。

## 四、问题2：仅知当天天气的单人在线决策

1. 题目未直接给出未来天气概率时，不得擅自声称存在唯一“期望最优策略”。分别研究：
   - 分布已知时的有限时域 MDP；
   - 分布未知时的鲁棒 MDP/分布鲁棒优化；
   - 强调不失败时的机会约束或风险度量（失败概率、最坏情形收益、CVaR）。
2. 明确天气是在当天行动前揭示，策略必须是“观测到当天状态后”的反馈策略，而不是偷看未来天气的开环路径。
3. 对第三关、第四关给出可执行的状态反馈规则，包括何时直奔终点、何时绕矿、何时停止挖矿、何时补给。
4. 对晴朗/高温/沙暴概率、天气转移相关性和风险偏好做敏感性分析。至少比较直接终点、矿山路线、村庄补给路线和鲁棒策略。
5. 用蒙特卡洛样本外评估报告：平均收益、中位数、5%分位数、失败率、最坏收益和置信区间。训练/调参用天气样本与最终评估样本必须分离。

## 五、问题3：多人竞争与信息结构

### 3.1 第五关：天气全知、day 0 一次承诺

1. 将每名玩家的完整行动方案视为纯策略，收益由路线拥堵、矿山收益分摊和村庄涨价共同决定。
2. 对两名玩家的第五关构造有限博弈。先通过可行策略生成/列生成压缩策略空间，再计算纯策略纳什均衡；若不存在或不唯一，再求混合策略均衡。
3. 报告收益矩阵或其稀疏表示、最佳反应、均衡、社会最优、价格无序（PoA）以及对风险偏好的敏感性。
4. 不能假设玩家可以在 day 0 后协调或修改计划。

### 3.2 第六关：仅知当天、行动后观察、三人动态博弈

1. 建立有限时域随机动态博弈/马尔可夫博弈。当天行动前只能使用公共历史和自身状态；当天结束后才能看到其他玩家当天行动和剩余资源，再决定下一天。
2. 讨论可计算的均衡概念：马尔可夫完美均衡、序贯均衡近似、相关均衡或多智能体强化学习近似。说明选择理由和局限。
3. 给出三人可执行策略，例如拥堵规避、矿山进入/退出阈值、对他人剩余资源的信念更新、随机化路线选择。
4. 用大量仿真比较自利均衡、合作基准、朴素同路策略和风险规避策略；报告个体收益、总收益、公平性、失败率和策略稳定性。

## 六、验证、消融与论文交付

1. 至少做四类验证：手算小例、规则单元测试、独立模拟回放、精确求解器与近似算法交叉验证。
2. 做消融实验：去掉村庄、禁止挖矿、忽略拥堵、忽略终点残值、改变负重、改变天气分布，解释结论变化。
3. 列出常见错误：把箱数与质量混用、沙暴行走、到矿当天挖矿、采购前允许负库存、多人同向边识别错误、把全知解用于在线问题、只给平均收益不报失败风险。
4. 最终输出应包括：
   - 300—500 字执行摘要；
   - 题意与假设；
   - 六关参数表和邻接表；
   - 三个问题的模型、算法与复杂度；
   - 第一、二关完整逐日结果和 Result.xlsx；
   - 第三至六关策略、实验和敏感性分析；
   - 图表清单与每张图的论文用途；
   - 局限性与可推广性；
   - 可运行代码目录、运行命令、随机种子、依赖版本和数据来源；
   - 可直接改写成竞赛论文的章节大纲。
5. 对“已由题面确认”“由模型假设”“由数值实验观察”“由外部文献支持”使用不同措辞。若证据不足，明确说不知道，并提出最小补充实验，不得编造。
