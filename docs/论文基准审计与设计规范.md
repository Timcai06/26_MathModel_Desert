# 论文基准审计与设计规范

## 1. 审计对象与结论

本规范以同一工作区内两篇已完成论文为对照：

- 《霍尔木兹海峡封锁对国际原油价格影响的动态建模与情景预测》，A4，57 页；
- 《基于混合整数规划、均值-CVaR 与相关市场审计的农作物种植策略研究》，A4，25 页。

审计同时检查 PDF 渲染页、正文结构和 LaTeX 源码。第一篇的优势是机制解释、资料分层、敏感性和附录证据充分，缺点是正文过长，部分页面表图密集且图中文字偏小。第二篇的优势是三问共享一条证据链、篇幅克制、样本外审计和局限披露清楚，缺点是黑白视觉语言较传统，关键结论没有形成贯穿全文的视觉索引。

本篇不以“页数更多”作为高级，而以“单位页面能让评审更快完成判断”作为高级。目标是保留第一篇的证据厚度和第二篇的审计纪律，同时建立更统一的地图、状态、策略、风险和博弈视觉系统。

## 2. 内容质量基线

### 2.1 主文结构

主文目标为 24--28 页，附录单独延伸；若赛制另有限页要求，以赛制为准压缩。正文采用以下固定叙事：

1. 摘要：逐问给出“方法—最优/推荐策略—关键数值—可信边界”；
2. 问题重述与难点：只保留决定建模结构的规则，不复抄题面；
3. 统一建模框架：图网络、资源流、信息集和收益函数一次定义；
4. 问题一：确定天气下的时间扩展网络 MILP、最优性证书和独立回放；
5. 问题二：天气先揭示的有限时域 MDP、稳健策略搜索和独立样本外评估；
6. 问题三：多人外部性、纯策略均衡/最佳响应和在线反馈博弈；
7. 综合检验：可行性、最优性或均衡偏差、分布漂移、敏感性与消融；
8. 评价、结论与可证伪条件；
9. 附录：逐日流水、算法伪代码、补充诊断和复现索引。

### 2.2 摘要验收

摘要控制在约 700--900 个汉字，必须出现：

- 第一、二关最终资金 `10470` 与 `12730`，并注明为 HiGHS 全局最优且经独立模拟器回放；
- 第三关概率假设、支撑内 1024 条天气序列零失败以及核心策略；
- 第四关最终入选策略的独立样本外失败率、均值和 5% 分位数；
- 第五关的均衡结构、单边偏离增益上界；
- 第六关的多人在线策略、相对基线的收益与尾部风险；
- 一句边界：题面未给未来天气分布，问题二和第六关的“最优”均为条件结论。

不写“结果良好”“具有一定参考价值”等无法审计的句子。

### 2.3 证据等级

每项结论标注其证据类型：

| 等级 | 证据 | 可使用表述 |
|---|---|---|
| E1 | 求解器最优证书 + 独立回放 | 全局最优 |
| E2 | 全状态/全序列精确枚举 | 给定模型下精确最优 |
| E3 | 有限策略空间的最佳响应检验 | 纯策略均衡或无利可图单边偏离 |
| E4 | 独立随机种子样本外评估 + 区间 | 推荐策略、统计优势 |
| E5 | 手工情景或训练样本比较 | 候选策略，不作为最终结论 |

正文中的“最优、稳健、均衡、显著”必须分别对应 E1--E4 证据，不能混用。

## 3. 视觉系统

### 3.1 色彩与语义

采用克制的沙漠主题，并保证灰度打印仍可区分：

| 语义 | 屏幕色 | 灰度/线型备份 |
|---|---|---|
| 主结构、确定性模型 | 深夜蓝 `#17324D` | 深色实线 |
| 可行/安全策略 | 绿洲青 `#168C8C` | 实线 + 圆点 |
| 不确定性/置信带 | 沙金 `#D6A64A` | 浅灰填充 + 斜线 |
| 风险/失败/拥挤 | 赭红 `#C45A3C` | 虚线 + 三角标记 |
| 背景网格 | 暖沙白 `#F6F0E6` | 5%--8% 灰 |
| 正文与坐标 | 墨黑 `#1B1F23` | 黑色 |

颜色只承担语义，不承担唯一编码；所有曲线同时使用线型或标记区分。连续色带使用感知均匀色图，禁止彩虹色图。

### 3.2 图形清单

正文只允许进入能直接支撑结论的图，候选图保存在 `output/candidate_figures/`，获正文引用后才复制到 `paper/figures/`。

计划中的正文主图：

1. 六关统一地图与规则卡片：同一坐标语言展示起点、终点、村庄、矿山和最短路；
2. 统一技术路线：信息集从“全知—当日知—多人反馈”逐级扩展；
3. 问题一时间扩展网络与资源—现金守恒示意；
4. 第一、二关逐日轨迹仪表板：位置、库存、现金、天气四轨对齐；
5. 第三关 MDP 状态转移与高温概率敏感性；
6. 第四关收益—失败率 Pareto 前沿及样本外置信区间；
7. 第五关双人路线时空图与最佳响应矩阵；
8. 第六关三人交互时空图、拥挤外部性分解和策略比较；
9. 六关证据总览：最优性/均衡/样本外验证矩阵。

每张图只回答一个标题式问题。主图宽度不小于 `0.82\textwidth`；双栏子图中任何坐标轴有效宽度不小于 70 mm；最终 A4 页面上最小文字不小于 8 pt。

### 3.3 表格规范

- 只用三线表，不使用竖线；
- 数值按小数点对齐，百分数和货币单位写在表头；
- 训练/选模与样本外结果分表或显式分栏；
- 最优值用粗体，推荐值用浅色底纹，失败值用警示符号；
- 表注交代样本数、随机种子、置信水平和失败样本处理口径；
- 正文表不超过 8 列，宽表移入横向附录或拆分。

## 4. 版式规范

- `ctexart` + XeLaTeX，A4；左右边距 24--25 mm，上下 22--24 mm；
- 中文正文 10.5--11 pt，行距约 1.30--1.36，不采用第二篇偏松的 1.42 倍行距；
- 一级标题居中或左对齐只能择一并贯穿全文；本篇采用左对齐深夜蓝标题和细沙金引导线；
- 页眉仅保留短标题与当前一级章节，页脚居中页码；摘要首页不显示页眉；
- 公式按“定义—解释—约束作用”组织，禁止连续堆叠无解释公式；
- 图题在下、表题在上，均写成可独立理解的结论性短语；
- 关键结论使用窄边框结论框，每个一级章节最多一个，防止视觉噪声；
- 不在正文放长代码，只保留 10--18 行核心伪代码，完整实现以文件索引进入附录。

## 5. 最终验收门槛

### 数值一致性

- 论文、JSON、CSV、Excel 和测试断言中的关键数值一致；
- 所有策略均经独立模拟器逐日回放；
- 问题一报告求解器状态和目标值，问题三报告最佳响应偏离增益；
- Monte Carlo 明确区分选模种子与最终样本外种子，并给出区间而非只报点估计。

### 内容完整性

- 三个问题、六关均有一般模型、具体策略、数值结果和边界；
- 每个模型至少有一个直接基线和一个消融/敏感性检验；
- 参考文献优先使用动态规划、随机控制、鲁棒优化、博弈论和数值优化的一手教材或论文；
- 所有外部事实有引用，题面事实引用原题，不伪造来源。

### PDF 视觉验收

- 编译零错误；未定义引用、越界盒和字体替代逐项清零或说明；
- 全文逐页渲染，检查空白页、孤行、标题压底、浮动体跨节和图中文字；
- 抽查 100% 表格列宽、公式编号和交叉引用；
- 灰度打印预览仍能区分全部策略；
- 最终 PDF 元数据、书签、页码、附件文件名和仓库复现命令一致。
